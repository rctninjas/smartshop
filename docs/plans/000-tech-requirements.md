## Цель

Зафиксировать обязательный технологический baseline для реализации админ-панели.

## Scope

- Только требования к технологическому стеку и организационным правилам UI-слоя админки.
- Без детального описания бизнес-флоу каталога/категорий/заказов.

## Ключевые решения

- Framework админ-панели: `Next.js 16`.
- UI library: `shadcn/ui`.
- ORM/data access для API: `Prisma 7`.
- Локальный запуск и dev-окружение выполняются через Docker.
- База данных (`PostgreSQL`) запускается в Docker-контейнере.
- Production-домены и SSL настраиваются через `HestiaCP` (вне контейнеров).
- Для dev-доступа достаточно `IP:порт` без доменных имен.
- Язык пользовательского интерфейса (`admin` и `storefront`): русский.
- Технические API-сообщения (`error.message` в backend-контрактах): английский.

## Docker baseline

- Используется `docker compose` как основной способ запуска локальной среды.
- Минимальный набор сервисов:
  - `admin` (Next.js app),
  - `api` (Fastify app),
  - `db` (PostgreSQL).
- Все сервисы поднимаются одной командой (`docker compose up`) в согласованной сети.
- Конфигурация окружения сервисов задается через `.env`/`.env.example`.
- В production reverse proxy и маршрутизация входящего трафика выполняются на стороне `HestiaCP`.
- Внутри docker-compose не используется отдельный ingress-контроллер.

## Cursor Skills

### Обязательно

- `vercel-react-best-practices` — практики React/Next.js по структуре и производительности.

### Рекомендуется

- `create-rule` — фиксация правил использования `shadcn/ui` в проекте.
- `create-skill` — создание проектного skill для повторяемых UI-паттернов админки.

## Правила использования UI

- Базовые UI-компоненты берутся из `shadcn/ui` и переиспользуются через `packages/ui`.
- Бизнес-логика не размещается в UI-компонентах (только presentation).
- Перед добавлением нового паттерна проверяется возможность переиспользования существующего.
- Все видимые пользователю тексты в `apps/admin` и `apps/storefront` должны быть на русском языке.

## Out of scope

- Конкретные API-контракты и модель данных доменов.
- Экранные сценарии конкретных модулей (каталог/категории/заказы).
