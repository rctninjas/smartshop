## Цель

Убрать MVP-ощущение интерфейса и привести админку к операционному UX-уровню: единые паттерны экранов, чистые формы, явные состояния и предсказуемая обратная связь для всех доменов (`catalog`, `categories`, `orders`, `customers`).

## Scope

- Полный UX-проход по `apps/admin` для основных модулей.
- Единый UX baseline на базе `shadcn/ui`-подхода и переиспользуемых компонентов.
- Без изменения бизнес-контрактов API.

## Этапы

### 1) UX-фундамент

- Вынести общие UI-кирпичи (`EmptyState`, `StatusBadge`, `Notice`, таблицы/toolbar-паттерны).
- Привести shell, page actions, формы и таблицы к единому визуальному языку.
- Подготовить удобные состояния `loading/empty/error/success`.

### 2) Catalog UX

- Улучшить таблицу: читаемые статусы, быстрые действия, понятные CTA.
- Улучшить формы create/edit: секции, подсказки, сохранение/ошибки.
- Минимизировать работу с «сырым JSON» в frequent path.

### 3) Categories UX

- Улучшить дерево и переходы по узлам.
- Привести schema editor к понятному draft/publish flow с индикаторами.

### 4) Orders + Customers UX

- Понятная работа со статусами заказа и деталями.
- Улучшение карточки клиента и навигации между сущностями.

### 5) Feedback и качество

- Убрать немые редиректы, добавить понятные уведомления.
- Провести smoke-проверки ключевых пользовательских сценариев.

## Критерии готовности

- В UI нет грубых MVP-паттернов и «немых» ошибок.
- Все ключевые экраны имеют консистентные состояния и обратную связь.
- Единые компоненты переиспользуются между модулями.
- Тексты интерфейса остаются на русском языке.
