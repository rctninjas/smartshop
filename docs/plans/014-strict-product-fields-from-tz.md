## Цель

Обеспечить строгое соответствие проекта исходному ТЗ по полям товара: все обязательные поля должны быть представлены в `DB + API + admin + docs`.

## Зафиксированные решения

- `price` — текущая цена.
- `sale` — цена со скидкой (отдельное поле).
- Правило «разные цвета — отдельные товары» фиксируется как процессное правило контент-ведения (без жёсткого backend-ограничения).

## Критерии готовности

- Поле `sale` присутствует в `Prisma`, `types`, `catalog API`, `admin UI`.
- Валидация: `sale >= 0` и `sale <= price`.
- Документация синхронизирована с реализацией и исходным ТЗ.
